## 五、烧录步骤

### 步骤 1：烧录发送端

1. 连接 ESP32 #1 到电脑
2. 打开 `lora_sender_demo.ino`
3. 选择开发板：`工具` → `开发板` → `ESP32 Dev Module`
4. 选择端口：`工具` → `端口` → `COMx`
5. 点击"上传"按钮
6. 烧录成功后，打开串口监视器（波特率 115200）

**预期输出：**
```
╔════════════════════════════════════════╗
║   LoRa Sensor Node - Demo Version     ║
║   校园微农园智能监测系统 (演示版)       ║
╚════════════════════════════════════════╝

Initializing LoRa...
LoRa initialized!
Frequency: 433.00 MHz
SF: 9
Bandwidth: 125.00 kHz
TX Power: 17 dBm

System initialized!
Send interval: 10 seconds

Waiting to send first packet...
```

### 步骤 2：烧录网关

1. 连接 ESP32 #2 到电脑
2. 打开 `lora_transmit（LoRa转发）.ino`
3. **可选**：修改 WiFi 配置（如果需要连接真实 WiFi）
4. 选择开发板和端口
5. 点击"上传"
6. 打开串口监视器（波特率 115200）

**预期输出：**
```
=== LoRa Gateway ===
Initializing LoRa...
LoRa initialized!
Connecting to WiFi: ...
```

### 步骤 3：开始演示

1. 确保两块板都通电
2. 观察发送端串口：每 10 秒发送一次数据
3. 观察网关串口：显示接收到的数据包

## 九、常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| LoRa 初始化失败 | 接线错误 | 检查 NSS、RST、DIO0 引脚 |
| 无法接收数据 | 频率不匹配 | 确认两端都是 433MHz |
| 接收数据乱码 | 参数不一致 | 检查 SF、BW、CR 设置 |
| WiFi 连接失败 | 配置错误 | 可跳过 WiFi，仅演示 LoRa |
| 串口无输出 | 波特率错误 | 设置为 115200 |
